# Настройка алертов Polymarket

## 🔔 Параметры алертов

Polymarket-алерты настраиваются на **двух уровнях**: глобально (дефолты для всех событий) и индивидуально (переопределения по конкретному событию). Эта страница описывает оба.

{% hint style="info" %}
**Правило наследования.** При добавлении нового события оно наследует текущие **глобальные** настройки. Per-event переопределения применяются поверх и никогда не влияют на глобальный пресет.
{% endhint %}

## 🌐 Глобальные параметры Polymarket

`Main Menu → Settings → Tracking Settings → Polymarket` <!-- VERIFY: точный путь меню для глобальных Polymarket-настроек -->

Глобальная панель содержит:

* **🔔 Частоту алертов** — `Real-time` / `Throttled` / `Digest`. Real-time пингует на каждое значимое событие; Digest группирует обновления в периодические сводки.
* **📈 Дефолтный порог движения цены** — сдвиг котировок, после которого срабатывает price-алерт. Ниже = шумнее, выше = тише.
* **💰 Дефолтный whale-порог** — минимальный USD-эквивалент сделки, считающейся «whale»-алертом.
* **📊 Алерты на всплеск объёма** — тоггл вкл/выкл; порог — `% прирост к среднему за 1 час`.
* **🟢 Алерты на новых трейдеров** — тоггл вкл/выкл; показывает первичных участников рынка.
* **🔕 Расписание тишины** — тихие часы, в которые Polymarket-алерты подавляются.

## 📝 Настройки конкретного события

Откройте `/polymarket → нажмите ✏️ рядом с нужным оддсом`. Панель редактирования — одна клавиатура из 7 строк, по кнопке на параметр:

* **🔈 Notifications** — мастер-тоггл ON/OFF для этого оддса. Иконка кодирует состояние: **🔈** = включено, **🔇** = выключено.
* **📉 Price Change** — срабатывает, когда цена оддса сдвигается на заданный процент.
* **🚨 Price Target** — срабатывает, когда цена пересекает абсолютное значение (в ¢).
* **♻️ Swaps Alert** — срабатывает на share-swap-сделках на рынке.
* **📊 Volume Target** — срабатывает, когда совокупный объём пересекает заданный USD-порог.
* **🗑 Delete** — прекращает трекинг этого оддса.
* **↩ Back** — назад к watchlist.

Тап по любой из 4 кнопок-порогов открывает value picker; текущее значение видно прямо в подписи кнопки после двоеточия (например, `📉 Price Change: 5%`).

## 🧪 Рекомендуемые пресеты

* **Наблюдение / низкий шум** — Digest, движение цены ≥ 10 п.п., whale ≥ $5 000. Получите несколько сгруппированных обновлений в день на событие.
* **Активный трейдер** — Real-time, движение цены ≥ 3 п.п., whale ≥ $1 000. Частые пинги; подходит для рынков, на которых вы активно играете.
* **Охотник за китами** — Real-time, price-алерты выключены, whale ≥ $25 000. Приходят только крупные сделки, независимо от движения котировок.

{% hint style="warning" %}
**Снижение порогов на сильно ликвидных рынках может затопить чат.** Крупные рынки вроде выборов могут генерировать десятки whale-сделок в час. Начинайте консервативно и ослабляйте постепенно.
{% endhint %}

## ❓ FAQ

<details>
<summary>Почему нет алертов, хотя рынок двигается?</summary>

Чаще всего движение не пересекло **порог движения цены**, либо сделки не пересекли **whale-порог**. Снизьте пороги глобально или переопределите по событию.

</details>

<details>
<summary>Можно ли полностью отключить Polymarket-алерты, не удаляя события?</summary>

Да — переведите глобальную частоту в **Off** или замьютьте отдельные события. Трекинг сохраняется; только уведомления прекращаются.

</details>

<details>
<summary>Расписание тишины применяется per-event или глобально?</summary>

Глобальное расписание тишины покрывает все Polymarket-алерты сразу. Per-event mute — это отдельный, более простой ON/OFF-тоггл.

</details>

<details>
<summary>Что происходит при изменении глобальных порогов — обновятся ли существующие события?</summary>

Существующие события сохраняют свои **per-event переопределения**. События без переопределений немедленно используют новое глобальное значение.

</details>

## 🔧 Troubleshooting

| Проблема | Вероятная причина | Решение |
|---|---|---|
| Никаких алертов, хотя всё включено | Глобальная частота алертов = **Off** | Переключите в Real-time, Throttled или Digest |
| Часть событий алертят, часть — нет | Per-event переопределение мьютит их или ставит очень высокие пороги | Откройте панель редактирования события и сбросьте на глобальные |
| Слишком много алертов | Слишком низкие пороги | Поднимите price-move и whale-пороги; переключитесь на Digest |
| Алерты во время тихих часов | Расписание не применено / несовпадение часового пояса | Подтвердите тайм-зону в `Settings → Notifications` |
